# 商业化可行性分析

> 版本：v2.0 | 日期：2026-06-22 | 状态：MVP已上线，本地运行

## 一、目标用户画像与市场规模

### 目标用户画像

| 维度 | 描述 |
|------|------|
| 核心用户 | 正在求职的职场人士，特别是转型期职场人（建筑→AI/互联网、传统行业→数字化等） |
| 年龄范围 | 25-40岁 |
| 特征 | 有一定工作经验，清楚自己的能力和短板，需要工具辅助决策而非替代决策 |
| 痛点 | 海量JD筛选困难、简历与岗位匹配度未知、投递过程缺追踪、面试准备无方向 |
| 使用场景 | 每天晚上/周末集中搜索和投递，每次深度使用1-2小时 |

### 规模估算

| 指标 | 数值 |
|------|------|
| 中国每年活跃求职者 | 约4000万人（含主动+被动求职） |
| 线上招聘市场规模 | 超200亿元/年 |
| 跨行业转型求职者 | 约300-500万人/年（经济转型期加速增长） |
| AI辅助求职工具细分市场 | 新兴市场，直接竞品少于5个 |

### 目标用户细分

1. **转型期职场人（核心用户）**：建筑→AI、土木→互联网、传统设计→产品等，占60%
2. **普通职场跳槽者**：同一行业内寻求更好机会，占25%
3. **应届生/初级职场人**：首次求职或工作1-3年，占15%

## 八、风险与应对

| 风险 | 可能性 | 影响 | 应对策略 |
|------|--------|------|----------|
| 招聘平台API封锁 | 中 | 高 | 多平台冗余，不依赖单一平台 |
| 爬虫反检测升级 | 中 | 中 | Playwright持续维护，降级为手动导入 |
| AI输出质量不稳定 | 中 | 中 | 结构化prompt + 硬约束过滤 + 用户确认环节 |
| 竞品入场（大厂） | 低 | 高 | 快速积累用户和数据壁垒，垂直深耕转型人群 |
| 用户留存率低 | 中 | 中 | 投递追踪形成数据黏性，持续优化AI分析质量 |
| 付费转化困难 | 高 | 中 | 低定价+免费版核心功能可用，降低决策门槛 |
