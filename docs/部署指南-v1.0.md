# 求职助手Web · 部署指南

> 版本：v1.0 | 日期：2026-06-16 | 方案A（零成本）

---

## 架构概览

```
前端 → Vercel (免费)
后端 → Render (免费, 750h/月)
数据库 → Supabase (免费, 500MB PostgreSQL)
AI → Pinme API (已有Key, ~¥5/月)
数据源 → Himalayas/Remotive (免费公开API)
```

---

## 步骤1：Supabase 数据库

1. 注册 https://supabase.com（GitHub登录）
2. New Project → 名称: `jobhunt` → 区域: Singapore
3. Settings → Database → Connection string → URI
4. 复制连接串，格式：`postgresql://postgres:[PASSWORD]@db.[PROJECT].supabase.co:5432/postgres`
5. 保存此连接串，后续配置Render时使用

---

## 步骤2：Render 后端部署

1. 注册 https://render.com（GitHub登录）
2. New → Web Service → 连接GitHub仓库
3. 配置：
   - **Root Directory**: `12-求职助手Web/backend`
   - **Runtime**: Python 3
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `uvicorn main:app --host 0.0.0.0 --port $PORT`
   - **Plan**: Free
4. Environment Variables:
   ```
   DATABASE_URL=postgresql://postgres:[PASSWORD]@db.[PROJECT].supabase.co:5432/postgres
   PINME_API_KEY=your_actual_key
   PINME_BASE_URL=https://pinme.cloud
   PINME_PROJECT_NAME=lingjing-e0c1
   CORS_ALLOW_ORIGINS=https://your-app.vercel.app
   ```
5. Deploy → 等待构建完成
6. 记录后端URL：`https://[service-name].onrender.com`

**注意：** Render免费层15分钟无请求会休眠，前端已配置心跳每10分钟ping `/api/health`。

---

## 步骤3：Vercel 前端部署

1. 注册 https://vercel.com（GitHub登录）
2. Add New → Project → Import Git Repository
3. 配置：
   - **Root Directory**: `12-求职助手Web/frontend`
   - **Framework Preset**: Vite
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
4. Environment Variables:
   ```
   VITE_API_URL=https://[service-name].onrender.com/api
   ```
5. Deploy → 记录前端URL：`https://[project].vercel.app`

---

## 步骤4：更新CORS配置

回到Render后端，更新环境变量：
```
CORS_ALLOW_ORIGINS=https://[project].vercel.app
```
重新部署后端。

---

## 步骤5：验证

1. 访问前端URL，注册账号
2. 搜索岗位（测试autocli/Mock/国际API）
3. JD直推分析（测试Pinme AI）
4. 查看周报和统计

## 本地开发

```bash
# 后端
cd 12-求职助手Web/backend
cp .env.example .env  # 填入PINME_API_KEY
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

# 前端
cd 12-求职助手Web/frontend
cp .env.example .env.local
npm install
npm run dev  # 默认http://localhost:5173
```

本地开发无需Supabase，自动使用SQLite。生产环境通过`DATABASE_URL`环境变量切换到PostgreSQL。

## 故障排查

| 问题 | 原因 | 解决 |
|------|------|------|
| 后端首次响应慢 | Render冷启动30-60s | 等待或刷新 |
| AI分析失败 | PINME_API_KEY未配置 | 检查Render环境变量 |
| 数据库连接失败 | DATABASE_URL格式错误 | 确认Supabase连接串 |
| CORS错误 | 前端URL未加入CORS | 更新CORS_ALLOW_ORIGINS |
| 国际岗位为空 | API暂时不可用 | 检查网络，或使用Mock数据 |
