## 一键部署（推荐）

### 方式一：Render Blueprint（后端 + 数据库）

1. Fork 仓库到你的 GitHub
2. 登录 [Render](https://render.com) → New → Blueprint
3. 连接 GitHub 仓库，Render 自动读取 `backend/render.yaml`
4. 自动创建 Web Service + PostgreSQL 数据库
5. 手动设置环境变量 `PINME_API_KEY`

### 方式二：Vercel 一键部署（前端）

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new)

点击上方按钮，或：
1. 登录 [Vercel](https://vercel.com) → Add New → Project
2. 导入 GitHub 仓库
3. Root Directory 设为 `frontend`
4. 环境变量 `VITE_API_URL` 设为 Render 后端地址
5. Deploy

## 手动部署

### 步骤1：Render 后端

1. 注册 [Render](https://render.com)（GitHub 登录）
2. New → Web Service → 连接 GitHub 仓库
3. 配置：
   - **Root Directory**: `backend`
   - **Runtime**: Python 3
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `uvicorn main:app --host 0.0.0.0 --port $PORT`
   - **Plan**: Free
4. 创建 PostgreSQL 数据库：
   - New → PostgreSQL → Plan: Free
   - 创建后自动注入 `DATABASE_URL` 环境变量
5. 手动添加环境变量：
   ```
   PINME_API_KEY=74365a3135df4a...（你的实际Key）
   PINME_BASE_URL=https://pinme.cloud
   PINME_PROJECT_NAME=lingjing-e0c1
   CORS_ALLOW_ORIGINS=https://你的前端域名.vercel.app
   ```
6. Deploy → 等待构建完成
7. 记录后端 URL：`https://career-shift-api.onrender.com`

> **注意**：Render 免费层 15 分钟无请求会休眠，前端已配置每 10 分钟心跳 ping `/api/health`。

### 步骤2：Vercel 前端

1. 注册 [Vercel](https://vercel.com)（GitHub 登录）
2. Add New → Project → Import Git Repository
3. 配置：
   - **Root Directory**: `frontend`
   - **Framework Preset**: Vite
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
4. 环境变量：
   ```
   VITE_API_URL=https://career-shift-api.onrender.com/api
   ```
5. Deploy → 记录前端 URL

### 步骤3：更新 CORS

回到 Render 后端，更新环境变量 `CORS_ALLOW_ORIGINS` 加入 Vercel 域名，重新部署。

## CI/CD 自动化

项目已配置 GitHub Actions（`.github/workflows/deploy.yml`）：

- **Push 到 main 分支** → 自动触发
- **前端**：TypeScript 类型检查 → Vercel 自动部署
- **后端**：Python 语法检查 → Render Deploy Hook

### 配置 GitHub Secrets

在 GitHub 仓库 Settings → Secrets and variables → Actions 中添加：

| Secret | 说明 | 获取方式 |
|--------|------|----------|
| `RENDER_DEPLOY_HOOK` | Render 部署 Webhook URL | Render Dashboard → Service → Settings → Deploy Hook |
| `VERCEL_TOKEN` | Vercel API Token | Vercel → Settings → Tokens |
| `VERCEL_ORG_ID` | Vercel 组织 ID | Vercel → Settings → General |
| `VERCEL_PROJECT_ID` | Vercel 项目 ID | Vercel → Project → Settings → General |

## 本地开发

```bash
# 后端
cd backend
cp .env.example .env  # 填入 PINME_API_KEY
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

# 前端
cd frontend
npm install
npm run dev  # http://localhost:5173
```

本地开发使用 SQLite，无需配置数据库。生产环境通过 `DATABASE_URL` 自动切换 PostgreSQL。

## 故障排查

| 问题 | 原因 | 解决 |
|------|------|------|
| 后端首次响应慢 | Render 冷启动 30-60s | 等待或刷新 |
| AI 分析失败 | PINME_API_KEY 未配置 | 检查 Render 环境变量 |
| 数据库连接失败 | DATABASE_URL 未自动注入 | 手动创建 PostgreSQL 并绑定 |
| CORS 错误 | 前端 URL 未加入 CORS | 更新 `CORS_ALLOW_ORIGINS` |
| 国际岗位为空 | API 暂时不可用 | 检查网络，或使用 Mock 数据 |
| 前端 404（SPA 路由） | Vercel 未配置 rewrites | 确认 `vercel.json` 存在 |
| Playwright 爬虫不可用 | Render 不支持浏览器 | 仅猎聘 MCP/Himalayas/Remotive 可用 |
