## 一键部署到 Hugging Face Spaces

### 步骤1：创建 Space

1. 打开 [huggingface.co/spaces](https://huggingface.co/spaces) → **Create new Space**
2. 配置：
   - **Space Name**: `jobhunt-assistant`
   - **License**: MIT
   - **Space SDK**: **Docker**
   - **Docker Template**: Blank
3. 点击 **Create Space**

### 步骤2：推送代码到 Space

创建完成后，HF 会显示 Git 地址。在终端执行：

```bash
cd /Users/wutianya/Desktop/我的AI工作系统/12-求职助手Web

# 添加 HF Space 作为远程仓库
git remote add space https://huggingface.co/spaces/你的用户名/jobhunt-assistant

# 推送代码
git push space main
```

### 步骤3：设置环境变量

1. 进入 Space → **Settings** → **Repository secrets**
2. 添加：

| 变量名 | 值 |
|--------|-----|
| `PINME_API_KEY` | 你的 Pinme API Key |
| `PINME_BASE_URL` | `https://pinme.cloud` |
| `PINME_PROJECT_NAME` | `lingjing-e0c1` |

3. 保存后 Space 自动重新构建

### 步骤4：等待构建完成

HF Spaces 会自动：
1. 安装 Node.js 依赖 → 构建前端（`npm run build`）
2. 安装 Python 依赖 → 启动 FastAPI（`uvicorn main:app --port 7860`）
3. 首次构建约 5-10 分钟

构建完成后，访问 `https://你的用户名-jobhunt-assistant.hf.space` 即可使用。

### 步骤5：验证

| 验证项 | 操作 |
|--------|------|
| 前端可访问 | 打开 Space URL，看到登录页 |
| 后端 API | 访问 `/api/health` 返回 `{"status":"ok"}` |
| 注册/登录 | 注册新账号 → 登录 → 进入工作台 |
| 岗位搜索 | 搜索「AI产品经理」→ 看到多平台结果 |

## 故障排查

| 问题 | 原因 | 解决 |
|------|------|------|
| Space 构建失败 | Dockerfile 语法错误 | 查看 HF Space 构建日志 |
| 前端无法访问 | nginx/静态文件路径错误 | 确认 Dockerfile 中前景构建产物路径正确 |
| 后端 API 500 | 数据库未初始化 | 检查 HF 构建日志，确认 `Base.metadata.create_all` 执行 |
| AI 分析失败 | PINME_API_KEY 未配置 | 检查 HF Space → Settings → Repository secrets |
| 注册/登录失败 | SQLite 权限问题 | 确认 /app/backend/data 目录可写 |
| 页面刷新 404 | SPA 回退未生效 | 确认 `PRODUCTION=true` 环境变量已设置 |
| 构建时间过长 | npm install 超时 | 重试 Space 构建 |
| 国际岗位为空 | Himalayas/Remotive API 不可用 | 自动降级为 Mock 数据，不影响使用 |

## 本地开发

```bash
# 后端
cd backend
cp .env.example .env  # 填入 PINME_API_KEY
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

# 前端
cd frontend
npm install
npm run dev  # http://localhost:5173
```

本地开发使用 SQLite，无需配置数据库。前端 Vite 自动代理 `/api` 到后端。
